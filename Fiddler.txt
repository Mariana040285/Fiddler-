Fiddler Everywhere - это высокопроизводительный кроссплатформенный прокси-сервер для веб-отладки для любого браузера, системы или платформы. Это инструмент веб-отладки, который регистрирует весь HTTP (S) трафик между вашим компьютером и Интернетом, проверяет и редактирует трафик, выдает запросы и обрабатывает входящие и исходящие данные.
Fiddler Everywhere регистрирует весь HTTP (S) трафик между вашим компьютером и Интернетом. Он захватывает трафик практически от любого приложения, поддерживающего системный прокси.

Ключевая особенность:
•	Совместим с MacOS, Windows и Linux.
•	Проверяйте веб-трафик (HTTP и HTTPS).
•	Расшифровать безопасный трафик (HTTPS).
•	Сохраняйте, делитесь и получайте захваченный трафик вместе с соавторами.
•	Имитация запросов и ответов.
•	Измените запросы и ответы.
•	Создавайте новые и редактируйте существующие запросы API.
•	Сохраняйте составленные запросы API и делитесь ими.
•	Расширенные возможности обмена и совместной работы.
•	Современный интуитивно понятный интерфейс с несколькими темами.
Определенная ключевая информация отображается в столбцах Live Traffic, включая:
•	# - Уникальный идентификационный номер, созданный Fiddler Everywhere.
•	Результат - код состояния ответа HTTP .
•	Протокол - Тип протокола (HTTP или HTTPS), используемый сеансом.
•	Хост - имя хоста и порт сервера, на который был отправлен запрос. В столбце также указаны запросы с методом CONNECT с параметром Tunnel to value (дополнительные сведения о туннелях CONNECT см. В Fiddler ).
•	Путь - путь относится к точному расположению страницы, публикации, файла или другого ресурса. Путь находится после имени хоста и разделяется символом «/» (косая черта).
•	URL-адрес - URL-адрес содержит протокол, имя хоста, порт и путь из запроса.
•	Метод - HTTP-метод, используемый в сделанном запросе (GET, POST, PUT и т. Д.).
•	Процесс - локальный процесс ОС, из которого исходит трафик (например, chrome: 8212 указывает на экземпляр браузера Google Chrome).
•	IP-адрес клиента - указывает IP-адрес клиента, отправившего этот запрос. В основном полезно, когда несколько компьютеров в сети указывают на один экземпляр Fiddler.
•	Удаленный IP-адрес - указывает IP-адрес сервера, используемого для этого запроса.
•	Размер тела - количество байтов в теле ответа.
•	Кэширование - значения из заголовков Response Expires и Cache-Control .
•	Content-Type - заголовок Content-Type из ответа.
•	Время - указывает время выполнения сеанса в формате ЧЧ: ММ: СС: мс .
•	Дата - указывает дату выполнения сеанса в формате ММ: ДД: ГГГГ .
•	Продолжительность - указывает продолжительность в миллисекундах, которая потребовалась для завершения сеанса.
•	Изменены правила - указывает, изменен ли сеанс активными правилами на вкладке « Правила ».
•	Комментарии Поле для отображения пользовательских комментариев, добавленных вами (или автором общего сеанса).
